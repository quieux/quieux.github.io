Sobre

Quando criei o gerenciador de pacotes QUEL, eu não tinha em mente criar uma distribuição. Utilizei-o no Arch Linux e funcionava muito bem. Caso eu tentasse utilizar um pacote QUEL em uma distro baseada no Debian, ele também funcionava bem na maioria das vezes. Isso porque, algumas vezes, eu tinha problemas de compatibilidade com as bibliotecas que cada distribuição utilizava. Então, surgiu a ideia de usar uma distribuição padrão para criar meus pacotes QUEL, pois eu teria a garantia de que os pacotes funcionariam e os únicos problemas seriam as incompatibilidades de hardware, que são mais fáceis de contornar.

A partir disso, procurei uma distro BLFS sem muita coisa instalada por padrão e encontrei a Exton BLFS. Usei-a como base e incluí o QUEL como seu gerenciador de pacotes. Assim surgiu o Quieux Linux, que não possui uma filosofia definida. O conceito é: Se está funcionando, mantenha. Se não está funcionando, corrija. Se não existe, crie.

Você não precisará baixar uma ISO nova depois de um tempo. Assim como no Arch, é possível atualizar o sistema sem reinstalá-lo completamente. Mas, ao contrário do Arch, você não terá atualizações constantes (a menos que você mesmo as faça). A estrutura e versão dos componentes serão mantidas até que isso comprometa a criação de novos pacotes ou tenha impacto de segurança para os usuários. Quando chegar o momento, a ISO para download no site será alterada, mas todos os usuários que já o tiverem instalado poderão atualizar usando o QUEL.

O QUEL também possui um conceito interessante de repositório padrão e repositório público. No repositório padrão, você encontrará pacotes de programas conhecidos, como o Firefox, que eu mesmo compilei. Mas o QUEL também oferece um repositório público utilizando o comando quel -pub. Nesse repositório público, serão aceitos pacotes dos mais variados tipos, criados pelos usuários, inclusive programas criados por eles mesmos. Desde que os pacotes não comprometam a segurança dos usuários, os pacotes públicos não terão qualquer restrição de criação. Por isso, deve-se ter cuidado ao utilizar qualquer um deles.
